# Лабораторная работа №2 (вариант 1): RSA

Java-приложение с графической формой (Swing) для:
- вычисления открытого ключа `Ko=(e,r)` по введенным `p`, `q`, `Kc=d`;
- побайтового шифрования произвольного файла по RSA;
- расшифрования файла, где каждый 16-битный блок хранит зашифрованный байт;
- демонстрации трех алгоритмов для отчета:
  - быстрое возведение в степень;
  - поиск первообразных корней (для `p=43`);
  - расширенный алгоритм Евклида.

## Структура

- `src/Main.java` — запуск GUI;
- `src/RSAFrame.java` — графическая форма (шифрование, расшифрование, алгоритмы);
- `src/RSAService.java` — RSA-шифрование/дешифрование файлов;
- `src/MathUtils.java` — математика (НОД, `modPow`, расширенный Евклид, обратный элемент);
- `src/PrimitiveRootFinder.java` — поиск первообразных корней;
- `src/AlgorithmShowcase.java` — печать шагов алгоритмов для отчета;
- `test-files/input.txt` — тестовый файл.

## Сборка и запуск (GUI)

Из корня проекта:

```powershell
javac -encoding UTF-8 -d out src\*.java
java -cp out Main
```

## Ограничения и проверки в программе

- `p`, `q` — простые и разные;
- `r = p*q` должно удовлетворять: `256 <= r <= 65535` (чтобы байт шифровался и помещался в 16 бит);
- `Kc=d`: `1 < d < phi(r)` и `gcd(d, phi(r)) = 1`;
- при расшифровании каждый восстановленный блок должен попадать в диапазон байта `[0..255]`.

## Пример теста

1. Запустить программу.
2. На вкладке `Шифрование RSA` ввести:
   - `p=257`
   - `q=251`
   - `Kc=17`
   - входной файл `test-files/input.txt`
   - выходной шифро-файл `test-files/encrypted.bin`
3. На вкладке `Расшифрование RSA` ввести:
   - `r=64507`
   - `Kc=17`
   - входной `test-files/encrypted.bin`
   - выходной `test-files/decrypted.txt`
4. Сравнить `input.txt` и `decrypted.txt` (должны совпасть).
